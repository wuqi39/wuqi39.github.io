idea中自动生成 import 语句

发表于 2024-12-16 更新于 2025-05-16 分类于 编程 本文字数： 414 阅读时长 ≈ 2 分钟
处理一些idea中自动生成 import 语句的问题

以下是调整后的标题分级版本（遵循Markdown标准，采用二级+三级标题结构，序号连续）：

一、IDEA中自动生成Import语句的设置与快捷键

1.1 设置自动导入工具提示

要在IDEA中显示自动导入的工具提示信息，可按以下步骤操作：

  1. 打开IDEA，点击菜单栏 File -> Settings。
  2. 在设置窗口中，选择 Editor -> General -> Auto Import。
  3. 勾选 Show the auto-import tooltip 选项。
  4. 点击 Apply 和 OK 保存设置。

1.2 自动导入类快捷键

  • Windows/Linux：Ctrl + Alt + Space
  • macOS：Command + Option + Space
    功能：输入类名时自动提示导入相关类。

1.3 手动导入类快捷键

  • Windows/Linux：Alt + Enter
  • macOS：Option + Enter
    功能：对未正确导入的类手动触发导入提示。

1.4 一键导入所有依赖

  • Windows/Linux：Ctrl + Alt + O
  • macOS：Command + Option + O
    功能：一次性导入代码中所有缺失的依赖类。

二、问题：自动生成长串局部注解


现象：使用Lombok等注解时，IDE生成@lombok.Getter而非import lombok.Getter;，导致代码冗余。

三、解决方法：优化导入配置

3.1 调整导入规则

  1. 打开IDEA设置（File -> Settings）。
  2. 选择 Editor -> Code Style -> Java -> Imports。
  3. 在 Class count to use import with ‘*’ 中输入较大数值（如999），减少通配符导入。
  4. 在 Name count to use static import with ‘*’ 下拉菜单中选择 *** for Lombok annotations**。
  5. 确保未勾选 Use fully qualified class names（完全限定类名）。

3.2 效果示例

配置后，使用Lombok注解时将自动生成简洁导入： 

import lombok.Getter;  // 而非@lombok.Getter  
@Getter  
public class Example { ... }
